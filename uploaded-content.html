图片SRC拦截与动态修改

前端实现图片资源拦截与替换的多种方法

实现方法

1. MutationObserver 监听DOM变化

监听DOM树变化，当有新的img元素添加时，修改其src属性。

// 创建MutationObserver实例 const observer = new MutationObserver((mutations) => { mutations.forEach((mutation) => { if (mutation.type === 'childList') { mutation.addedNodes.forEach((node) => { if (node.tagName === 'IMG') { // 拦截并修改图片src interceptImage(node); } }); } }); }); // 开始观察 observer.observe(document.body, { childList: true, subtree: true });

2. 重写Image构造函数

重写全局Image构造函数，拦截所有通过new Image()创建的图片。

// 保存原始Image构造函数 const OriginalImage = window.Image; // 重写Image构造函数 window.Image = function() { const img = new OriginalImage(); // 拦截src属性设置 let _src = ''; Object.defineProperty(img, 'src', { get: function() { return _src; }, set: function(value) { // 修改图片URL _src = modifyImageUrl(value); img.setAttribute('src', _src); } }); return img; };

3. 拦截HTMLImageElement的src属性

通过修改HTMLImageElement原型上的src属性，拦截所有img元素的src设置。

// 获取原始src属性描述符 const originalDescriptor = Object.getOwnPropertyDescriptor(HTMLImageElement.prototype, 'src'); // 重写src属性 Object.defineProperty(HTMLImageElement.prototype, 'src', { get: function() { return originalDescriptor.get.call(this); }, set: function(value) { // 修改图片URL const modifiedUrl = modifyImageUrl(value); originalDescriptor.set.call(this, modifiedUrl); } });

4. Service Worker拦截网络请求

通过Service Worker拦截图片网络请求，实现更底层的资源替换。

// 在Service Worker中 self.addEventListener('fetch', (event) => { if (event.request.destination === 'image') { event.respondWith( // 修改图片请求URL或返回替代图片 fetch(modifyImageUrl(event.request.url)) ); } });

注意：Service Worker需要HTTPS环境，此处仅作演示。
